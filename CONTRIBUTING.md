## 🚀 How to Contribute

### 1. Reporting Issues
Before opening a new issue, please check the [Issue Tracker](https://github.com/Vigilan-sec/vigilan/issues). When reporting a bug, please provide:
* Your environment (OS, Network setup).
* Relevant logs or error messages.
* Steps to reproduce the issue.

### 2. Submitting a Pull Request (PR)
1. **Fork** the repository.
2. Create a **new branch** for your feature or fix (`git checkout -b feature/amazing-new-rule`).
3. **Commit** your changes with clear and descriptive messages.
4. **Push** your branch to your fork.
5. Open a **Pull Request** against our `main` branch.

---

## 🛠️ Code Standards & Security

* **Security First:** Since Vigilan is a security tool, ensure your code follows best practices. Avoid common vulnerabilities like buffer overflows or unsanitized inputs.
* **Testing:** If you add a new detection signature, please provide a sample **PCAP** file or a specific command to trigger the alert.
* **Documentation:** Update the `README` if your change adds a new configuration flag or a major feature.
